# 夏日消暑祭 抽獎網站 部署說明

架構：前端（GitHub Pages）＋ 後端（Google Apps Script）＋ 資料庫（Google Sheets）。
依序完成以下四個步驟即可上線。

## 步驟一：設定 Google Sheets

1. 到 [Google Sheets](https://sheets.google.com) 建立新試算表，命名如「夏日消暑祭抽獎」。
2. 建立三個分頁（名稱必須完全一致）：`訂單資料`、`抽獎紀錄`、`獎池庫存`。
3. 各分頁第 1 列填入標題列：
   - **訂單資料**：`客戶編號`｜`訂單號碼`｜`訂單金額`
   - **抽獎紀錄**：`抽獎時間`｜`客戶編號`｜`訂單號碼`｜`訂單金額`｜`第幾次抽獎`｜`中獎獎項`｜`獎品內容`｜`獎品類型`
   - **獎池庫存**：`獎項名稱`｜`初始數量`｜`已抽出數量`｜`剩餘數量`
4. 在「獎池庫存」分頁第 2～7 列填入（獎項名稱含 emoji，需與下表完全一致）：

   | 獎項名稱 | 初始數量 | 已抽出數量 | 剩餘數量 |
   |---|---|---|---|
   | 🍧 刨冰獎 | 3 | 0 | 3 |
   | 🍵 抹茶獎 | 8 | 0 | 8 |
   | 🍡 糰子獎 | 22 | 0 | 22 |
   | 🧊 冰沙獎 | 30 | 0 | 30 |
   | 🥤 蘇打獎 | 45 | 0 | 45 |
   | 🫧 彈珠汽水獎 | 59 | 0 | 59 |

## 步驟二：部署 Google Apps Script

1. 在試算表中點選 **擴充功能 → Apps Script**，開啟編輯器。
2. 刪除預設內容，貼入本專案 `gas/Code.gs` 的完整程式碼，儲存。
3. 點右上角 **部署 → 新增部署作業**：
   - 類型選 **網頁應用程式**
   - 「執行身分」選 **我**
   - 「誰可以存取」選 **所有人**
4. 按「部署」，第一次會要求授權——選擇你的 Google 帳戶並允許。
5. 複製取得的 **網頁應用程式 URL**（形如 `https://script.google.com/macros/s/XXXX/exec`）。

> ⚠️ 之後若修改 Code.gs，需要「部署 → 管理部署作業 → 編輯 → 新版本」才會生效。
>
> ⚠️ 活動期間固定為 `2026/07/16 00:00` ～ `2026/08/14 23:59`（台北時區）。若需調整檔期，
> 修改 `gas/Code.gs` 開頭的 `ACTIVITY_START` / `ACTIVITY_END` 常數後重新部署新版本。

## 步驟三：部署前端到 GitHub Pages

1. 到 GitHub 建立新 Repository（例如 `summer-lottery`，Public）。
2. 編輯 `js/config.js`，把步驟二取得的 URL 貼入：
   ```js
   API_URL: 'https://script.google.com/macros/s/XXXX/exec',
   ```
3. 推送本專案到 GitHub：
   ```bash
   git remote add origin https://github.com/<你的帳號>/summer-lottery.git
   git push -u origin main
   ```
4. 到 Repository 的 **Settings → Pages**：Source 選 `Deploy from a branch`，Branch 選 `main` / `(root)`，按 Save。
5. 約 1 分鐘後取得網址：`https://<你的帳號>.github.io/summer-lottery/`。

## 步驟四：開始使用

1. 在「訂單資料」分頁輸入客戶訂單，例如：`C001`｜`ORD-20260716-001`｜`9500`。
2. 開啟抽獎網址，輸入上述客戶編號＋訂單號碼測試一次完整抽獎。
3. 確認「抽獎紀錄」出現一筆紀錄、「獎池庫存」對應獎項剩餘數量 -1。
4. 將抽獎網址分享給會員；後續在 B/C 分頁即時查看中獎結果與庫存。

## 常見問題

- **前端顯示「獎池載入失敗」**：檢查 `js/config.js` 的 API_URL 是否正確、GAS 是否部署為「所有人可存取」。
- **修改獎項數量**：直接改「獎池庫存」分頁的數字即可，前端會即時反映。
- **活動結束**：把「獎池庫存」剩餘數量全部改為 0，前端會顯示「活動獎品已全數抽出」；或等到 2026/8/14 過後，系統會自動依日期拒絕抽獎並顯示「活動已結束」。
- **8/14 之後客戶仍可查詢／抽獎嗎？**：不行，`gas/Code.gs` 會自動檢查活動期間（`ACTIVITY_START`/`ACTIVITY_END`），期間外一律回覆活動未開始或已結束，即使獎池仍有庫存。
